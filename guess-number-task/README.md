### Описание проекта

Угадать загаданное компьютером число за минимальное число попыток.

:arrow_up:[к оглавлению](_)

### Какой кейс решаем?

Нужно написать программу, которая угадывает число за минимальное число попыток

**Условия соревнования:**

- Компьютер загадывает целое число от 0 до 100, и нам его нужно угадать. Под «угадать», подразумевается «написать программу, которая угадывает число».
- Алгоритм учитывает информацию о том, больше ли случайное число или меньше нужного нам.

**Метрика качества**
Результаты оцениваются по среднему количеству попыток при 10000 повторений

**Что практикуем**
Написание хорошего кода на python

:arrow_up:[к оглавлению](_)

### Краткая информация о данных

Генератор случайных чисел задает последовательность чисел.
Алгоритмы, используемые проектом, угадывают каждое загаданное число из списка.
Оценивается среднее количество попыток для решения этой задачи разными алгоритмами.
  
:arrow_up:[к оглавлению](.README.md#Оглавление)

### Этапы работы над проектом

Анализ базовых алгоритмов, имеющихся в задании.
Подбор оптимального алгоритма решения задачи.
Для решения использованы 2 варианта алгоритма бинарного поиска (№№ 3 и 4).
В варианте 3 новое значение угадываемого числа принимается за половину динамического интервала поиска (min_predict и max_predict).
В варианте 4 - новое значение угадываемого числа случайно выбирается внутри динамического интервала поиска (min_predict и max_predict).

:arrow_up:[к оглавлению](.README.md#Оглавление)

### Результаты

Оба предложенных алгоритма обеспечивают выполнение целевого параметра, указанного в заданнии - а именно поиск решения менее, чем за 20 попыток.

Вариант №3 в cреднем решает задачу угадываения числа за 5 попыток, вариант №4 - за 7 попыток.

:arrow_up:[к оглавлению](.README.md#Оглавление)
